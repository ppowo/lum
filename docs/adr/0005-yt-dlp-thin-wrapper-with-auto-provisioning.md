# yt-dlp Thin Wrapper with Auto-Provisioning

Lum's `yt` subcommand shells out to `yt-dlp` as a thin wrapper, constructing args for three user-facing commands (`aud`, `vid`, `alb`) that map to audio, video, and album/playlist downloads. yt-dlp is auto-provisioned: if not found on `$PATH`, lum downloads the latest release binary to `data_dir()/deps/yt-dlp` with co-located state in `data_dir()/deps/yt-dlp.json` (version string + last-checked timestamp). The auto-provisioned copy is re-checked against the GitHub latest release no more than once per 24 hours; if a newer version exists, lum replaces it. The system copy on `$PATH` is never version-policed. aria2c is dropped entirely — yt-dlp's native `--concurrent-fragments 8` replaces it since YouTube uses DASH/HLS fragments where aria2c's per-file split settings were ineffective. ffmpeg remains a hard prerequisite, not auto-provisioned, because it is too large and environment-dependent. Output directories are set via yt-dlp's `-P` flag rather than `chdir`. Video quality defaults to `bestvideo[height<=1080]+bestaudio/best` with an optional `--height` flag instead of probing display resolution. Six `--replace-in-metadata` regex passes clean YouTube title junk. There is no default rate limit, no `--embed-*` post-processing, and no `--tmp` debug flag.
