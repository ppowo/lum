# YouTube Radio Uses yt-dlp and PATH ffmpeg

YouTube-backed `lum radio` stations are treated as normal built-in radio stations at the CLI layer, but they use a separate media path from direct audio streams. Direct stations continue to use the pure Rust stack from ADR 0002: reqwest, Symphonia, CPAL, ringbuf, and rubato. YouTube station page URLs are first resolved with yt-dlp, reusing the existing yt-dlp auto-provisioning from ADR 0005. The resolved media URL may be an HLS playlist rather than a direct audio container; for this path, lum invokes `ffmpeg` from `$PATH` and asks it to emit stereo `f32le` PCM at the active output sample rate, which is then written to the existing CPAL audio buffer.

For now, ffmpeg is not auto-provisioned. Missing ffmpeg is reported as a runtime requirement for YouTube radio stations only. Static ffmpeg provisioning can be added later as a shared dependency resolver once platform sources, archive layout, update policy, and licensing posture are decided. This avoids blocking YouTube live radio support on packaging decisions while keeping direct radio streams independent of ffmpeg.
